FAQ

What’s the purpose of MonetizeJS?

MonetizeJS provides user payment services for applications that are not tied to any server or that don’t want to implement user management.

How can I perform payments using MonetizeJS

Payments are performed via redirections to the MonetizeJS platform. You have 2 ways to perform a redirection:

  • Using the monetize.js cliend-side library, you can redirect the user from your application (full-page redirection or popup) using simple JavaScript calls.
  • Using the OAuth2 flow, you can redirect the user to the authorize endpoint.

How can I check payments from my application

Payments can be verified from your application by:

  • Using the monetize.js cliend-side library, you can validate payments in a pure client-side application.
  • Using the REST API, you can validate payments from a client-side application or from your server, using an access token retrieved from the monetize.js cliend-side library or the OAuth2 flow.
  • Using the CouchDB API, you can check particular user’s payments from your server.

What are the requirements for my application to use MonetizeJS?

MonetizeJS platform requires an HTML5 compatible environment to handle redirections, including modern browsers (Chrome, Firefox, IE9+, Safari…), hybrid mobile applications, HTML5 based desktop applications…

How much does it cost?

MonetizeJS applies a 5% fee on top of the Stripe processing fees.

How are users impacted by pricing option modifications?

When a developer modify a pricing option, users that already have paid are not impacted, which means that:

  • current once-off charge is evaluated to its original price.
  • current subscription will continue on its original plan.

Can I convert a subscription to a once-off charge?

No, once-off charges and subscriptions are not convertible. However, a developer can freely credit user’s total charges.
