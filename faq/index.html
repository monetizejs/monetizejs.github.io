FAQ

What’s the purpose of MonetizeJS?

MonetizeJS provides user payment services for applications that aren’t tied to any server or that don’t want to implement user management.

How much does it cost?

As a seller, MonetizeJS applies a 5% fee on top of Stripe’s processing fees. MonetizeJS doesn’t charge a fee to use the service as a buyer, but fees may be charged by your bank in connection with the debiting of your card.

How do I cancel a subscription?

To cancel a subscription, go to “My apps”, select the application and click “Unsubscribe”. The subscription will remain active until the end of the billing period, at which point it won’t be renewed.

How can I perform payments using MonetizeJS?

Payments are performed via redirections to the MonetizeJS platform. You have 3 ways to perform the redirection:

  • Using the monetize.js client-side library, you can redirect the user from your application (full-page redirection or pop-up) using simple JavaScript calls.
  • Using the OAuth2 flow, you can redirect the user to the authorize endpoint.
  • Using the MonetizeJS button.

How can I check payments from my application?

Payments can be verified from your application by:

  • Using the monetize.js client-side library, you can validate payments in a pure client-side application.
  • Using the REST API, you can validate payments from a client-side application or from your server, using an access token retrieved from the monetize.js client-side library or the OAuth2 flow.
  • Using the CouchDB API, you can check a particular user’s payments from your server.

What are the requirements for my application to use MonetizeJS?

MonetizeJS platform requires an HTML5 compatible environment to handle redirections, including modern browsers (Chrome, Firefox, IE9+, Safari…), hybrid mobile applications, HTML5 based desktop applications…

How are users impacted by pricing option modifications?

When a developer modifies a pricing option, users that have already paid are not impacted, which means:

  • current one-off charge is evaluated at its original price.
  • current subscription will continue to be charged at its original rate.
